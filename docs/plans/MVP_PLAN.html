Referencia de Diseño — evamuah.com

Análisis del sistema de diseño de evamuah.com para replicar en shadcn/ui + Tailwind CSS v4. Sitio premiado en Awwwards (Mención Honorífica, 2022). Estética: minimalismo editorial con sensibilidad de marca de lujo.

Paleta de Colores

Fondo principal#F6F3EECrema cálido (no blanco puro)
Texto principal#2B2626Marrón oscuro (no negro puro)
Texto secundariorgba(43,38,38,0.8)80% opacidad
Superficie secundaria#F3ECE6Linen (crema más cálido)
Acento decorativo#ECDFD4Rosa suave
Acento durazno#FBCFC6Peach (elementos decorativos)
Navegaciónrgba(246,243,238,0.85)Efecto vidrio esmerilado

Clave: todo es cálido y tonal. Sin contrastes agresivos. Crema en vez de blanco, marrón en vez de negro.

Tipografía

Títulos (display)Rfceqabro DisplaySerif editorial de alto contraste. Sustituto: Playfair Display o Cormorant Garamond
Cuerpo / UIInterGoogle Fonts, pesos 300-700
Hero (h1)~22rem / weight 400Masivamente grande
Subtítulo (h2)~4rem / weight 400
Cuerpo~1rem / weight 400Inter, line-height 1.5
Botones~0.8rem / uppercaseInter, weight 400

Clave: contraste tipográfico extremo — serif gigante para títulos, sans-serif pequeño para cuerpo. El tamaño hace el trabajo, no el peso.

Espaciado y Bordes

  • Contenedor: max-width ~1128px con padding lateral de 5rem
  • Secciones: padding vertical de 5rem a 12rem (muy generoso)
  • Botones: pill shape (border-radius: 900px), borde 1px sólido, fondo transparente
  • Imágenes: sin border-radius (bordes rectos), object-fit: cover
  • Sin sombras, sin gradientes, sin botones rellenos
  • Divisores: línea 2px sólida color texto

Animaciones

  • Intro: letras del nombre animadas individualmente (fade/slide secuencial)
  • Panel: overlay de pantalla completa que se desliza para revelar contenido
  • Scroll: fade up (opacity 0→1, translateY arriba) al entrar en vista
  • Texto: revelación escalonada (stagger)
  • Paralaje: imágenes con escala o desplazamiento en scroll
  • Navegación: transparente → sólida al desplazar

Decisiones Adicionales

📁
Convex como apps/convex/ — El folder de Convex funciona como proyecto raíz, no como subcarpeta de backend/. La estructura es apps/convex/schema.ts, no apps/convex/convex/schema.ts.
🔄
Schema diseñado para estado final — Se incluye la tabla subscriptions desde el inicio (plan, status, provider, periodos) aunque la integración de pagos recurrentes sea posterior. Evita migraciones disruptivas.
🤖
Auto-traducción con Mistral API — Al crear contenido de blog o cursos, un botón «Traducir» genera automáticamente la versión EN a partir del ES (o viceversa) usando Mistral. Incluye control de uso (rate limits, costo por token, indicador de consumo mensual).
📧
Resend preconfigurado sin dominio — Toda la lógica de correo transaccional queda implementada y probada con el dominio de prueba de Resend. Se activa con dominio real cuando se adquiera.
📦
Datos mock desde el inicio — La plataforma arranca con contenido de ejemplo (cursos, lecciones, textos de landing, testimonios, imágenes placeholder). Katherin los modifica desde el panel admin. Sin necesidad de migración de datos.
🎬
Estudio de costos de video realizado — Ver docs/research/VIDEO_STREAMING_COSTS.html para comparativa detallada de Bunny Stream vs Mux vs Cloudflare vs R2 autoalojado con costos por escala. Resultado: se construyó y adoptó devultur (R2 autoalojado, SaaS de media propio) — integración completa en 0.6.11 (refs tipadas, upload+transcode, session binding).

Fases de Implementación

1 Andamiaje del Monorrepo ~2h ▶

Crear toda la estructura de carpetas, configurar dependencias y verificar que el servidor de desarrollo inicia correctamente.

  • packages/config/ — TSConfig base, React y Convex compartidos
  • packages/utils/ — Función cn(), formateadores de moneda (COP), fechas
  • packages/ui/ — shadcn con estilo base-lyra (Base UI), tema oscuro con acentos para marca de maquillaje. Instalar componentes base: Button, Card, Input, Dialog, Table, Sidebar, etc.
  • apps/convex/ — Proyecto Convex con schema inicial y auth config
  • apps/web/ — TanStack Start con:
    • vite.config.ts: devtools → tailwindcss → tanstackStart → viteReact
    • router.tsx: ConvexQueryClient + QueryClient + SSR integration
    • routes/__root.tsx: ConvexBetterAuthProvider, meta tags, estilos
    • lib/env.ts: validación de variables de entorno con @t3-oss/env-core
    • lib/auth-client.ts y lib/auth-server.ts
    • lib/i18n.ts: configuración i18next (ES por defecto)
  • Actualizar root package.json con scripts dev:web, dev:convex
  • Datos mock: seed inicial con cursos de ejemplo, lecciones, testimonios, textos de landing e imágenes placeholder. Todo editable desde el panel admin

    • Verificación: pnpm dev inicia ambos servidores. La página placeholder se renderiza en localhost:3000.

    Referencia: Replicar patrones exactos de /home/demar/projects/me-myself-and-i/mail-watcher/

    2 Autenticación ~3h ▶

    Registro, inicio de sesión y recuperación de contraseña. Distinción de roles admin/estudiante.

    • Convex: tabla users, funciones getByAuthId, updateProfile, setRole, blockUser
    • Better Auth: email/password + Google + Apple con plugin convexClient()
    • Ruta API: /api/auth/$ — handler catch-all (GET + POST)
    • Páginas: login, register, forgot-password con TanStack Form + Zod
    • Guards:
      • _authenticated.tsx — layout para rutas protegidas (redirige a login si no autenticado)
      • admin/_layout.tsx — verifica role === "admin" en beforeLoad
    • Resend: correo de bienvenida y restablecimiento de contraseña (bilingüe según locale del usuario). Se configura con dominio de prueba de Resend; se activa con dominio real cuando se adquiera
    3 Página de Marca Personal ~8h ▶

    Landing page tipo evamuah.com con estética editorial minimalista, paleta crema cálida y tipografía serif de alto contraste. Contenido mock inicial que Katherin modifica desde el admin.

    • Hero: video o imagen de fondo con overlay, texto animado (Motion animate + transition), CTA, indicador de scroll
    • Navegación: barra fija transparente → sólida al desplazar (useScroll + useTransform), toggle de idioma, botones auth
    • Portafolio: grilla de imágenes con hover zoom, animaciones escalonadas al entrar en vista (useInView)
    • Sobre mí: foto + biografía con animaciones de entrada
    • Testimonios: carrusel con contenido desde tabla siteContent (editable por admin)
    • Cursos destacados: 2-3 cursos destacados del catálogo, enlace a /courses
    • Pie de página: enlaces sociales, navegación rápida, selector de idioma
    • i18n: todo el texto visible usa t() de react-i18next
    • Responsive: menú hamburguesa en móvil, secciones apiladas, animaciones reducidas si prefers-reduced-motion
    4 Schema Completo + Admin Base ~4h ▶

    Desplegar el modelo de datos completo en Convex y construir la estructura del panel de administración.

    • Schema: crear todas las tablas (courses, lessons, purchases, progress, invitationLinks, siteContent, blogPosts) con índices
    • Admin layout: barra lateral con navegación agrupada:
      • Tablero (resumen general)
      • Contenido: Cursos, Artículos, Contenido del Sitio
      • Usuarios: Lista de usuarios
      • Ventas: Invitaciones
    • Tablero: tarjetas resumen (total usuarios, cursos, compras recientes, estudiantes activos) con datos en tiempo real de Convex
    5 Gestión de Cursos (Admin) ~6h ▶

    CRUD completo de cursos y lecciones con carga de video a devultur.

    • Convex functions: courses.create, .update, .updateStatus, .reorder, .delete; mismo patrón para lessons.*
    • devultur (video):
      • useMediaUpload (@devultur/convex/react) — una máquina de estado upload → transcode; dispara processVideo solo y borra el video viejo en replace
      • Al completar, guarda la ref DevulturMedia en lessons.video
      • Estado del video vía useVideoState (unión discriminada: queued → processing → ready | failed)
    • Páginas admin: tabla de cursos con acciones (editar, lecciones, publicar/archivar), formulario de creación/edición con campos bilingües, gestión de lecciones con reordenar (arrastrar), carga de video con indicador de progreso
    • Imágenes: <CropUpload> de devultur (recorte a aspecto fijo → WebP) para miniaturas de cursos y portadas
    • Invitaciones: generar código único, configurar máximo de usos y expiración, copiar enlace
    6 Catálogo de Cursos (Público) ~4h ▶

    Navegación pública de cursos con detalle, vista previa y control de acceso.

    • Listado: grilla de tarjetas (miniatura, título, extracto, precio, cantidad de lecciones). Responsive: 1→2→3 columnas
    • Detalle: hero con miniatura/video preview, descripción completa, temario (lista de lecciones con duración, íconos candado/libre), botón «Comprar» o «Continuar viendo»
    • Acceso: función access.hasAccess(userId, courseId) verifica: admin → suscripción activa → compra completada → lección gratuita
    • Invitación: ruta /courses/invite/$code valida código, si autenticado redime y redirige; si no, pide login primero
    7 Reproductor de Video + Progreso ~6h ▶

    Reproductor HLS con seguimiento de progreso por segundo, avance automático y reanudación.

    • Reproductor: <VideoPlayer> de devultur (hls.js interno, token-transparente con session binding). Controles: reproducir/pausar, buscar, volumen, pantalla completa, velocidad
    • Progreso:
      • Evento timeupdate del <video> (~4x/seg)
      • Debounce a mutación Convex cada 10 segundos
      • También envía en: pausa, cambio de visibilidad, desmontaje, finalización
      • Marca completed: true cuando watchedSeconds ≥ duration × 0.9
    • Hook: useProgressTracker(lessonId, courseId, duration) → retorna { onTimeUpdate, startAt }
    • Avance automático: overlay al completar «Siguiente: [Título]» con cuenta regresiva 5s, botones «Reproducir ahora» y «Cancelar»
    • Barra lateral: lista de lecciones del curso, lección actual resaltada, completadas con ✓, en progreso con barra, bloqueadas con 🔒
    • Reanudación: al navegar al curso, busca última lección no completada con watchedSeconds > 0 y la inicia desde ahí
    8 Gestión de Usuarios (Admin) ~3h ▶

    Administración de usuarios con búsqueda, bloqueo y gestión de acceso a cursos.

    • Listado: tabla con nombre, correo, rol, fecha de registro, estado. Búsqueda por nombre/correo
    • Detalle: perfil del usuario, lista de cursos adquiridos con progreso, acciones:
      • «Otorgar acceso a curso» (seleccionar curso, crea purchase con grantedBy: "admin")
      • «Revocar acceso» por curso
      • «Bloquear/Desbloquear usuario»
    9 Contenido del Sitio + Blog ~2h ▶

    Editor de contenido para la landing page y sistema de blog.

    • Editor de contenido: lista de claves con valores ES/EN lado a lado, edición inline, agrupado por sección (Hero, Sobre mí, Testimonios, Pie de página)
    • Landing dinámica: componentes de la landing consultan siteContent por clave, con fallback a strings de i18next si no hay contenido en DB
    • Blog admin: tabla de artículos, formulario de creación/edición con campos bilingües, publicar/despublicar, botón «Traducir» con Mistral API (genera traducción automática ES→EN o EN→ES con indicador de consumo y límites de uso)
    • Blog público: grilla de artículos publicados, página de detalle con meta tags SEO
    10 Analítica y Monitoreo ~1h ▶

    Integración de PostHog y Sentry en planes gratuitos.

    • PostHog: inicializar en __root.tsx, captura automática de vistas de página, eventos personalizados: course_viewed, lesson_started, lesson_completed, purchase_completed, invitation_redeemed
    • Sentry: inicializar con DSN, error boundary, carga de source maps en build

    Decisiones Técnicas Clave

    🎬
    devultur <VideoPlayer> (hls.js interno) — expone onProgress con el timeupdate del <video> para el seguimiento por segundo, sin el postMessage frágil de un iframe. Delivery privado (HLS + captions) con session binding transparente.
    ⏱
    Debounce de 10 segundos — Enviar progreso cada 10s equilibra precisión de reanudación con volumen de escrituras. A 100 estudiantes simultáneos: ~10 mutaciones/seg, dentro del plan gratuito de Convex.
    🌐
    Datos bilingües como { es, en } — Objetos directos en Convex en lugar de tabla de traducciones. Consultas simples, sin joins. Aceptable porque solo hay 2 idiomas.
    📝
    i18n híbrido — Strings estáticos de UI vía archivos JSON de i18next (rápido, sin consultas DB). Contenido dinámico (hero, testimonios) vía tabla siteContent (editable por admin).
    🎟️
    Invitaciones como registros de compra — Al redimir, se crea un purchases con grantedBy: "invitation" y amount: 0. La verificación de acceso solo consulta purchases.
    🖼
    devultur para todo el media — SaaS de media propio (R2 autoalojado). Imágenes públicas (miniaturas, portadas, landing — og-safe, tokenless) y video privado (HLS + session binding). Refs tipadas DevulturMedia vía dv.media() de @devultur/convex/validators, no strings de URL.
    🔒
    Auth guard en beforeLoad — TanStack Router ejecuta beforeLoad antes de renderizar. Se consulta el usuario, se verifica el rol y se lanza redirect() si no tiene permisos. Sin destello de contenido no autorizado.

    Verificación End-to-End

    • pnpm dev inicia servidores web y Convex simultáneamente
    • Landing page se renderiza en localhost:3000 con animaciones
    • Toggle de idioma cambia entre ES y EN en toda la página
    • Registro con correo electrónico → recibe correo de bienvenida vía Resend
    • Login con Google funciona y redirige correctamente
    • Rutas protegidas redirigen a login si no autenticado
    • Panel admin solo accesible para usuarios con role: "admin"
    • Admin crea curso → agrega lecciones → sube video → publica
    • Catálogo público muestra solo cursos publicados
    • Detalle de curso muestra temario con lecciones libres y bloqueadas
    • Admin genera enlace de invitación → estudiante lo redime → obtiene acceso
    • Estudiante reproduce lección → progreso se guarda cada 10s
    • Al cerrar y volver, la lección reanuda desde el último segundo
    • Al completar lección, avance automático a la siguiente con cuenta regresiva
    • Admin puede bloquear usuario → usuario pierde acceso
    • Admin edita contenido del sitio → cambios se reflejan en la landing
